# 贡献指南

欢迎参与 Hello-AI。

## 你可以做什么

- 新增章节或案例
- 修正文档错误
- 补充参考资料
- 优化图表、链接和结构

## 提交要求

- 内容尽量可复现、可验证
- 新术语首次出现时给出中文解释
- 涉及版本、模型、API 时注明来源和时间
- 图片、外链、示例文件要可追溯
- 改动尽量保持单一主题，方便审核和回滚

## 提交流程

1. 从主分支创建功能分支
2. 在本地修改 Markdown 内容
3. 预览并运行本地检查
4. 提交 PR
5. 等待审核后合并

本地检查命令：

```bash
python scripts/build_local.py
```

这个命令会检查导航完整性、Markdown 链接、资源引用，并执行严格构建。

## 页面标签

新增页面时，可以在 Markdown 顶部添加标签，便于后续做标签筛选和专题索引：

```yaml
---
tags:
  - Prompt
  - Lab
---
```

可用标签在 `mkdocs.yml` 的 `material/tags` 配置中维护。

## 写作风格

- 面向小白，但不装傻
- 先给路径，再讲原理
- 每章保留最小示例和常见错误
